# Working Preferences

## For planning
- Break work into clear phases and checklists
- When priorities conflict, finish current foundation before opening new modules
- For this repo, content quality matters more than flashy structure changes

## For repo operations
- Build should pass before declaring completion
- Do not redesign the full repo when the task is limited to one or two articles
- Source-of-truth hierarchy: xem `openspec/AGENTS.md` (Read Order section)

## For writing content
→ Xem **Writing Rules** trong `openspec/AGENTS.md` — đó là SOT cho quy tắc viết.
